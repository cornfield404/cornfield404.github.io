题目： DEEP THINK WITH CONFIDENCE
链接： 2508.15260
代码： GitHub - facebookresearch/deepconf: Code Release for Deep Think with Confidence
单位： Meta AI；UCSD

背景

对于 LLM 来说，测试时有一些后处理方法可以显著提高模型的准确率：

  • 生成多个答案，然后进行 voting。该方法会产生大量的额外计算开销，同时该方法是边际收益递减的；
  • 用答案中每个 token 的概率分布计算该答案的置信度，高置信度的答案通常对应着较低的熵。通过每个 token 的置信度和熵可以计算得到某个答案的全局置信度，过滤掉低置信度的答案可以提升 voting 的准确率。该方法的缺陷是：采用全局置信度可能导致某些关键的低置信度 token 被掩盖掉；同时，全局置信度需要生成完整的答案，同样会产生大量的额外计算开销。

推理质量的指标

Token Entropy

Hi=−∑jPi(j)log⁡Pi(j)H_i=-\sum_jP_i(j)\log P_i(j) Hi​=−j∑​Pi​(j)logPi​(j)

H 较低表明模型确定性高，反之，H 较高表明模型确定性低。

Token Confidence

Ci=−1k∑j=1klog⁡Pi(j)C_i=-\frac{1}{k}\sum_{j=1}^k\log P_i(j) Ci​=−k1​j=1∑k​logPi​(j)

C 较高表明模型确定性高，C 较低表明模型确定性低。

Average Trace Confidence

Cavg=1N∑i=1NCiC_{avg}=\frac{1}{N}\sum_{i=1}^NC_i Cavg​=N1​i=1∑N​Ci​

Confidence Measurements

Group Confidence

计算某个 sliding window 内的平均置信度。

CGi=1∣Gi∣∑t∈GiCtC_{G_i}=\frac{1}{|G_i|}\sum_{t\in G_i}C_t CGi​​=∣Gi​∣1​t∈Gi​∑​Ct​

Bottom 10% Group Confidence

取最低的 10%置信度计算平均，相当于关注某个答案中置信度最低的 10%有多低。

Cbottom−10(t)=1∣Gb∣∑Gj∈GbCGiC_{bottom-10}(t)=\frac{1}{|G_b|}\sum_{G_j\in G_b}C_{G_i} Cbottom−10​(t)=∣Gb​∣1​Gj​∈Gb​∑​CGi​​

Lowest Group Confidence

取整个回答的最低置信度，是 Bottom 10% Group Confidence 的一个极端情况。

Cleast(t)=min⁡Gj∈GCGjC_{least}(t)=\min_{G_j\in G}C_{G_j} Cleast​(t)=Gj​∈Gmin​CGj​​

Tail Confidence

计算整个回答最后部分的置信度。因为 LLM 通常会把分析放在中间，结论放在最后，尤其这篇文章实验里主要用的数学数据，所以该指标相当于更多关注结论的置信度，而不是思维链中间过程的置信度。

Ctail(t)=1∣Ttail∣∑t∈TtailCtC_{tail}(t)=\frac{1}{|T_{tail}|}\sum_{t\in T_{tail}}C_t Ctail​(t)=∣Ttail​∣1​t∈Ttail​∑​Ct​

离线方法

离线方法里，不同的答案是事先生成好的，需要做的就是把不同答案聚合得到最终结果。

Majority Voting

直接投票，出现次数多的答案为最终答案。

Confidence-Weighted Majority Voting

计算每个答案的 confidence 的加权。

Confidence Filter

根据每个答案的 confidence，先按照一定比例，选出 confidence 最高的答案，然后再根据 Confidence-Weighted Majority Voting 选出最终答案。

在线方法

在线方法里，可以在答案生成的过程中计算答案的置信度，并且直接停掉那些置信度低于阈值的答案，以节约资源并加快结果生成速度。

Offline Warmup

  • 该操作是用来计算阈值的，计算好了之后 online 生成的结果低于该阈值就会被 kill 掉；
  • s=Percentile100−η({Ct:t∈Twarmup})s=Percentile_{100-\eta}(\{C_t:t\in T_{warmup}\})s=Percentile100−η​({Ct​:t∈Twarmup​})
  • 具体方法是对于一个 prompt，先生成一小批答案（数量远少于正式计算的时候），然后根据这一小批答案的 confidence 确定阈值。

Adaptive Sampling

  • 根据问题的难易程度决定生成答案的数量；
  • 问题的难易程度由已经生成的答案之间的一致性决定；
  • 一致性的计算由投票出的答案的权重除以所有答案的权重之和计算：β=V(a^)∑aV(a)\beta=\frac{V(\hat{a})}{\sum_aV(a)}β=∑a​V(a)V(a^)​，这个值比较大表示答案之间的一致性较高，问题比较容易；
  • 当 β\betaβ 较低时，会继续生成答案；当 β\betaβ 比较高时，根据当前的答案生成最终结果。

个人总结

整体来说是一个对大模型部署进行的改进，设计了一些置信度计算的方法筛选最终的结果。但是我感觉这个方法只能用在一些回答比较固定的任务上，如果 answer 发散，现在这些投票的方法应该都需要重新设计，因为目前的方法 voting 的操作看起来就是直接比较两个答案一不一样，对于大模型来说这个要求应该还挺严格的。

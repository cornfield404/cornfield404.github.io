【DeepSeek系列（一）】DeepSeek LLM

发表于2025-03-18|更新于2025-04-24|技术积累论文笔记
|总字数:1.7k|阅读时长:6分钟|浏览量:

题目： DeepSeek LLM Scaling Open-Source Language Models with Longtermism
链接： 2401.02954
代码： GitHub - deepseek-ai/DeepSeek-LLM: DeepSeek LLM: Let there be answers
单位： DeepSeek-AI

本文目的

研究了语言模型中的 scaling 问题，之前的论文要么忽略了 scaling 问题，要么对于该问题研究不够充分，本文希望得到一个更可靠的结论。

具体来说，本文先研究了随着模型大小变化，batch size 和 learning rate 的 scaling law。在此基础上，对模型和数据的 scaling law 进行了全面研究，并据此预测了大模型的表现。此外，在开发过程中，还发现了在不同数据集上会得到不同的 scaling law。

Pre-Training

Data

  • 数据构建原则是增强数据集的丰富性和多样性；
  • 具体来说，将数据处理分为三个步骤：去重、过滤和 remix，其中去重和 remix 保证数据多样性，过滤阶段通过增加信息密度，使模型更有效地训练；
  • 采用 BBPE 构建 token；
  • 将数字拆分，分为单独的数字；
  • 常规 token 有十万个，在 24G 的多语言语料库上进行了生成，特殊 token 15 个，最终在训练时把 vocabulary 大小设为了 102400。

Architecture

  • Pre-norm structure、RMSNorm function、SwiGLU、RoPE、67B 模型采用了 Grouped-Query Attention；
  • Optimizer：AdamW，β1=0.9,β2=0.95\beta_1=0.9,\beta_2=0.95β1​=0.9,β2​=0.95，weight_decay=0.1；
  • Learning rate 采用 multi step learning rate，2000 step warm up，训练完成 80%，lr 降为最大值的 31.6%，训练完成 90%，lr 降为最大值的 10%；
    • 选用 multi step 主要是因为与 cosine 相比性能相当，且便于实验调试；
  • Gradient clip 设为 0.1。

Infrastructures

  • HAI-LLM，包括 data parallelism、tensor parallelism、sequence parallelism 以及 1F1B pipeline parallelism；
  • Flash attention；
  • ZeRO-1；
  • 在 fp16 上训练，但梯度累计在 fp32 上；
  • VLLM

Scaling Laws

当计算能力提升时，应该增大模型规模还是增大数据量，以达到最好的模型效果是一个需要研究的问题。

Scaling Laws for Hyperparameters

  • 为了保证之后在模型和数据规模上进行实验时，每组实验都能取得最佳性能，因此预先研究在不同的计算量下超参数的 scaling law，后面的实验就按照这个 scaling law 取超参；
  • 实验现象：
    • 泛化误差对于 batch size 和 learning rate 不是特别敏感，在一定范围内都能取得较优的效果；
    • 随着计算预算的增加，最佳 batch size 逐渐增大（Bopt=0.2920⋅C0.3271B_{opt}=0.2920\cdot C^{0.3271}Bopt​=0.2920⋅C0.3271），最佳 learning rate 逐渐降低（ηopt=0.3118⋅C−0.1250\eta_{opt}=0.3118\cdot C^{-0.1250}ηopt​=0.3118⋅C−0.1250）；
    • 在相同计算预算的情况下，模型大小和数据规模的不同分配对超参的选择仅有轻微影响。

Estimating Optimal Model and Data Scaling

  • 在一定计算量的情况下，找到模型大小与数据量的最佳分配方式；
  • 在本文中数据量由数据集中的 token 数来表示；
  • 本文定义的模型规模由 non-embedding FLOPs/token 表示，论文中采用的公式为 M=72nlayerdmodel2+12nlayerdmodellseqM=72n_{layer}d_{model}^2+12n_{layer}d_{model}l_{seq}M=72nlayer​dmodel2​+12nlayer​dmodel​lseq​，但论文中没有给出计算方法，以下是我自己推的：
    • 模型 backward 计算量约为 forward 的两倍，可参考 2001.08361，因此只要计算出 forward 计算量，然后乘 3 就可以；
    • Forward 过程中不考虑 embedding 过程，仅考虑 attention+FFN，加法和乘法各算一次；
    • Attention 过程包括 key、value 和 weight 的 project， attention map 计算，attention map 与 value 加权，output project：
      • key、value 和 weight 的 project 是矩阵乘法，由 ntoken×dinn_{token}\times d_{in}ntoken​×din​ 的矩阵乘以 din×dattnd_{in}\times d_{attn}din​×dattn​ 大小的矩阵，计算量为：3×2×ntoken×din×dattn3\times 2\times n_{token}\times d_{in}\times d_{attn}3×2×ntoken​×din​×dattn​；
      • Attention map 由 key 和 value 相乘得到，计算量为：2×ntoken2×dattn2\times n_{token}^2\times d_{attn}2×ntoken2​×dattn​；
      • Attention map 与 value 加权，计算量为：2×ntoken2×dattn2\times n_{token}^2\times d_{attn}2×ntoken2​×dattn​；
      • Output project 计算量为：2×ntoken×dattn×dout2\times n_{token}\times d_{attn}\times d_{out}2×ntoken​×dattn​×dout​；
    • FFN 过程的计算量包括两次 project，计算量为：4×ntoken×dFFN×dout4\times n_{token}\times d_{FFN}\times d_{out}4×ntoken​×dFFN​×dout​；
    • 在通常设置里，attention 模型的输入、输出及中间维度均为 dmodeld_{model}dmodel​，FFN 中间维度为 4×dmodel4\times d_{model}4×dmodel​，汇总即可得到论文里给的公式。
  • 设置不同的计算预算，给每个计算预算设置了约 10 个不同的模型数据分配方式，超参由上一节公式计算，并在验证集上计算泛化误差；
  • 根据实验得到：Mopt=0.1715⋅C0.5243,Dopt=5.8316⋅C0.4757M_{opt}=0.1715\cdot C^{0.5243},D_{opt}=5.8316\cdot C^{0.4757}Mopt​=0.1715⋅C0.5243,Dopt​=5.8316⋅C0.4757，在后续试验中验证了小规模模型上的实验可以比较准确地预测大模型的泛化误差。

Scaling Laws with Different Data

当数据集质量增高时，最优分配策略会倾向于在模型上分配更多资源（即增大模型规模）。

Alignment

数据

采用了约 150 万个中英文指令数据实例，涵盖了广泛的有益和无害主题。其中有益数据包含 120 万个实例，31.2%用于一般语言任务，46.6%用于数学问题，22.2%用于编码练习。安全数据由 30 万个实例组成，涵盖了各种敏感主题。

SFT

  • 在 7B 模型上用 learning rate 1e-5 训练了 4 个 epoch，在 67B 模型上用 learning rate 5e-6 训练了 2 个 epoch。
  • 在 SFT 过程中，随着数学数据量的增加，模型发生复读机的概率在变大，在小模型中会更明显；论文中推测是因为数学数据中可能会包含一些比较相似的模式，较弱的模型没办法掌握这种模式，就导致产生了复读。

DPO

  • 在不同领域收集了多语言 prompt，并用 deepseek chat 模型生成候选回答；
  • Batch size 为 512，learning rate 为 5e-6，训练 1 个 epoch；
  • 实验发现，DPO 可以增强模型的 open-ended generation skill。

Evaluation

  • DeepSeek 在英语测试集上表现与 LLaMA2 相当，虽然两个模型训练数据量大致相同，但 DeepSeek 用的是双语语料；
  • DeepSeek 67B 相对于LLaMA2 70B 的优势大于 DeepSeek 7B 相对于LLaMA2 7B 的优势，可能是由于多语料冲突在小模型中表现更明显。

个人总结

这篇主要的重点还是在于 scaling law，模型整体设计没有什么特殊的地方。

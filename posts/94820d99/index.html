题目： DeepSeekMath: Pushing the Limits of Mathematical Reasoning in Open Language Models
链接： 2402.03300
代码： GitHub - deepseek-ai/DeepSeek-Math: DeepSeekMath: Pushing the Limits of Mathematical Reasoning in Open Language Models
单位： DeepSeek-AI；Tsinghua University；Peking University

本文要点

  1. 通过精心设计的数据处理过程，充分利用网络上的公开数据；
  2. 设计了 GRPO 替代 PPO。

预训练数据收集

数据收集流程

  1. 把 OpenWebMath 作为 seed corpus。从 seed corpus 里随机抽 50w 数据作为证样本，从 Common Crawl 里随机抽 50w 数据作为负样本，训练一个 fastText model；
  2. 在 Common Crawl 去重后的数据上运行 fastText model，找到数学相关的数据。为了保证筛出的样本质量，筛选数据时会根据 fastText model 预测的 score 对数据排序，仅保留分值较高的数据。在第一轮迭代里，仅保留前 40B tokens；
  3. 根据 base URL 把数据分为不同的 domain。统计每个 domain 的数据在第一轮迭代被分为数学相关的比例，超过 10%的就认为这个 domain 是数学相关的。然后，手工对这些 domain 里与数学相关的部分进行标注。链接到这些 URL 但没在数据集里的网页也被加入 seed corpus；
  4. 收集了更多数据后，重新训练 fastText 得到一个更好的模型，在下一轮里召回更多与数学相关的网页；
  5. 4 轮之后得到 35.5M 数学相关的网页，120B tokens；第四轮停止的原因是第四轮迭代新收集的数据量很少了；
  6. 采用了 DeepSeek_Coder 中 gram 过滤相同的方法过滤数据，防止测试数据进入训练集。

数据集质量评估


用相同的模型在不同数据集上训练相同的 token 数，用本文提出的数据集作为训练集在不同的下游任务上均取得了最好的效果。

预训练

实验设置

模型从 DeepSeek-Coder-Base-v1.5 7B 初始化，训练 500B token。训练数据构成：56%数学数据集，4% AlgebraicStack，10% arXiv，20% Github code，剩余 10%是来自 Common Crawl 的语言数据。
训练过程中 batch size 最大值为 4.2e-4，batch size 为 10M tokens。

预训练结果

预训练后的模型在多种数学任务，包括利用 python 代码解题以及进行证明的任务上都取得了很好的效果。

SFT

数据

构建了一个包含中英文的数学 instruction-tuning 数据集，每个问题对应的回答的形式为 CoT、PoT 或 tool-integrated reasoning format。数据集中数据数量为 776K。

训练设置

训练样本随机 concat 知道长度达到 4K tokens。Batch size 为 256，学习率为 5e-5，训练 500 个 step。

Group Relative Policy Optimization (GRPO)

PPO 的缺陷

  1. 需要的模型过多，需要 policy model，reference model，reward model 和 value model，训练时显存压力比较大；
  2. LLM 训练里面，通常会对等到最后一个 token 结束，对完整的输出进行打分。但是价值函数需要学到到每个 token 为止的 value，这是比较困难的。
    PPO 的详细推导可以参考强化学习

GRPO 的改进


GRPO 整体思路其实和 PPO 都差不多，主要的区别：

  1. PPO 采用 value model 和 reward model 计算每个 step 的 advantage，GRPO 采用 ri−mean(r)std(r)\frac{r_i-mean(r)}{std(r)}std(r)ri​−mean(r)​ 作为每个 step 的 advantage，也就是同一个 sequence 所有 token 的 advantage 是一样的；
  2. 第一步的计算方式过于简单，因此论文中也引入了其他计算方式，例如可以把回答分成若干个 step，每个 step 赋予一定的 reward（reward 同样需要归一化），每个 token 的 advantage 由 following steps 的 reward 相加得到；
  3. PPO 在计算 advantage 的时候引入了 policy model 和 reference model 的 KL divergency，GRPO 把这一项提出来了，简化了 advantage 的计算。

GRPO 训练参数

学习率 1e-6，KL coefficient 0.04，每个 question sample 64 个回复，max length 为 1024，batch size 1024。

个人总结

这篇主要工作有两点：

  1. 数据收集：利用网页本身的性质并采用循环优化分类模型和数据集的方式，以少量的标注代价收集到了一个大规模高质量的数学相关文本数据集；
  2. GRPO：与 PPO 在思路上基本上是一样的，只不过通过修改 advantage 的计算方式，省掉了一个 critic model 的模型。

题目： Qwen-VL: A Versatile Vision-Language Model for Understanding, Localization, Text Reading, and Beyond
链接： 2308.12966
代码： GitHub - QwenLM/Qwen-VL: The official repo of Qwen-VL (通义千问-VL) chat & pretrained large vision language model proposed by Alibaba Cloud.
单位： Alibaba Group

模型结构

LLM

Qwen-7B

Visual Encoder

ViT-bigG

Position-aware Vision-Language Adapter

该结构的目的

因为图像生成的 token 数量比较多，因此该结构会把图像的 token 数进行压缩。

具体做法

  • 采用一层随机初始化的 cross attention 结构，随机初始化的 embedding 作为 query，图像 feature 作为 key 和 value；
  • Embedding 数量为 256，因此图像 token 经过压缩后会变成 256 个；
  • Cross attention 结构的 query 和 key 都有 position embedding。

该部分的实现代码

模型输入输出

图像输入

图像经过 visual encoder 和 adapter 处理之后，得到了固定数量的 token。图像 token 的前后加入了<img>和</img>用于区分图像 token 和文本 token。

Bbox 的输入输出

  • 输入的 bbox 被 normalize 到 0～1000，格式是 (Xtopleft,Ytopleft),(Xbottomright,Ybottomright)(X_{topleft},Y_{topleft}),(X_{bottomright},Y_{bottomright})(Xtopleft​,Ytopleft​),(Xbottomright​,Ybottomright​)；
  • Bbox 前后加入<box>和</box>用于区分 bbox 和普通文本；
  • <ref>和</ref>用于标记描述 bbox 的文本。

模型训练

Pre-training

  • 该阶段训练采用了大规模、弱标注、网上爬下来的图像-文本对；
  • 清洗后的数据共 1.4B 个图像-文本对，其中 77.3%是英文文本数据，22.7%是中文文本数据；
  • 该阶段训练中，LLM 不参与训练，只有 ViT 和 adapter 参与训练；
  • 输入数据尺寸为 224×224224\times 224224×224；
  • Learning rate 的最大值是 2e−42e^{-4}2e−4；
  • Batch size 为 30720 个图像-文本对，共训练 50000 step，相当于 1 个 epoch 多点。

Multi-task Pre-training

  • 引入了质量更高的 VL 数据，在下列 7 个任务上进行了训练：
  • 在数据组织上，把同任务的数据放到同一个 sequence 里面，sequence 的长度为 2048；
  • 输入数据尺寸为 448×448448\times 448448×448；
  • 模型的所有部分都参与训练。

Supervised Fine-tuning

目的

增强对话能力。

具体做法

  • 数据一部分来源于 LLM 生成，一部分引入了手动处理（主要是一些更复杂的数据，比如 localization 以及多图数据）；
  • 训练时混合了纯文本数据和多模态数据；
  • 训练数据量为 350k；
  • 这步训练时冻结了 ViT，只训 LLM 和 adapter。

个人总结

这篇文章的 adapter 后来也被抛弃了，当时的想法应该是借鉴了 detr 类的想法，把图像的整体包括物体的信息具体信息通过 query embedding 提取出来。训练遵循了初步图文对齐->在具体的图像任务上面 finetune->sft 的过程。另外，这个模型应该是有多图能力的，但是在论文里面没提怎么处理的多图。

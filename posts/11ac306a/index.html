链接： 2501.12948
单位： DeepSeek-AI

DeepSeek-R1-Zero

Reward Modeling

R1-Zero 设计了一个完全基于规则的 reward 函数，主要包括两部分：

  • 结果是否正确：比如数学题的最终结果或者编程题的编译器反馈结果；
  • 格式是否正确：指是否在 <think><think><think> 和 </think></think></think> 之间输出了思考过程。

Aha Moment

#### 缺陷 DeepSeek-R1-Zero 的输出可能有多种语言混合且可读性比较差，比较难以直接应用。

DeepSeek-R1

Cold Start

  • 采用了少量 long CoT 的数据用于 fine-tune 模型，目的是让训练更稳定（相当于直接 RL 是需要模型自己摸索出来 think 的过程，先 sft 一下可以给模型一些 sample 让模型模仿，更容易训练）；
  • 同时，直接 RL 的话模型输出的可读性不够好，在 cold start 过程中，可以人为控制训练数据格式，使得模型的输出更易读。在这篇文章里，deepseek 提供的格式是 ∣special_token∣<reasoning_process>∣special_token∣<summary>|special\_token|<reasoning\_process>|special\_token|<summary>∣special_token∣<reasoning_process>∣special_token∣<summary>。

Reasoning-oriented Reinforcement Learning

  • 在一些逻辑性比较强的领域（例如：数学、coding、logic reasoning）进行 RL 训练，提升模型的 reasoning 能力；
  • 加入了 language consistency reward 帮助模型保证输出结果尽量是单一语言的。

Rejection Sampling and SFT

  • Cold start 的数据主要侧重于 reasoning，这一轮 sft 的数据包括一些其他通用领域；
  • Reasoning data：用上一步的模型生成的，对每个 prompt，生成多个 response，生成之后用一些格式上的规则过滤掉了一些明显不合理的答案，然后在剩下的答案里保留正确的（正确与否的评估按照我的理解有一部分是用规则评估的，还有一些需要用 reward model 评估）。总共收集了 60w 数据。
  • Non-Reasoning data：重用了 DeepSeek-v3 的 sft 数据集，对于部分数据，在问题回答前调用 DeepSeek-v3 生成了 potential CoT，对于比较简单的 query 没有生成 CoT。总共收集了 20w 数据。
  • 两部分加起来共有 80w 数据，这一步 sft 训了 2 个 epoch。

Reinforcement Learning for all Scenarios

  • 这一步是为了和人类偏好对齐；
  • 对于 reasoning data，采用 R1-Zero 的方法，利用基于规则的 reward 引导模型；
  • 对于 general data，采用模型评估人类偏好。对于 helpfulness 的部分，只关注 summary，不关注思考过程；对于 harmlessness 的部分，关注整个推理过程。

模型效果

个人总结

一个对于大模型里 RL 的探索，直接用 RL 的话在指标上也可以取得还不错的效果，但是在可读性上会比较差。用少量数据进行 SFT 可以克服该困难。

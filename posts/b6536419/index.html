Node-Limited Routing

  • 每个 token 最多被发到 MMM 个 node 上；
  • Node 的选择规则是把每个 node 最高的 KrM\frac{K_r}{M}MKr​​ 个分数相加，然后从里面选出相加总和最高的 M 个 node。

Token-Dropping Strategy

V3 没有采用（主要是解决了负载问题，不需要了）。

Multi-Token Prediction

采用 MTP 的优势

  • 相当于增加了监督，可以提高数据利用效率；
  • 因为这样相当于用更远的 token 监督当前 token，可以使模型更好地预测未来的结果。

MTP Modules

  • 用 DDD 个额外的 module 预测 DDD 个额外的 token；
  • 每一个 module 的 embedding layer 和 output head 是共享的；
  • 每个 module 会把上个 module transformer 的输出和本 module embedding layer 得到的 feature 拼到一起作为本 module transformer 的输入：

Infrastructures

训练架构

采用了 16 路 pipeline parallelism，64 路 expert parallelism 以及 ZeRO-1 data parallelism，除此之外还设计了一些新的架构优化训练，将在下面逐一介绍。

DualPipe and Computation-Communication Overlap

  • 引入该优化的原因：跨节点的 expert parallelism 引入的通信开销比较高；
  • 但是其实没有充分理解这个具体是怎么操作的，按照目前的理解，大概一方面是用了一个双向流水线，另一方面调整了用于计算和用于通信的资源比例。

Efficient Implementation of Cross-Node All-to-All Communication

  • 跨节点通讯靠 IB 处理，节点内通讯靠 NVLink（速度更快）；
  • 每个 token 最多分给 4 个 nodes，降低 IB 通信量。

Extremely Memory Saving with Minimal Overhead

  • 放弃保存 RMSNorm 以及 MLA up-projection 的结果，等到 bp 需要用到的时候重新计算；
  • Exponential Moving Average 的参数保存在 CPU 里；
  • 在物理层面上使得 MTP 的 embedding 层和 output head 共享。
  • 通过上面这些优化，可以使的 DeepSeek-v3 不需要使用 tensor parallelism。

FP8 训练

混合精度框架

  • 大多数计算都是在 FP8 精度上执行的，但是输出可能是 BF16 或 FP32；
  • Embedding module、output head、MoE gating module、normalization operators、attention operators 保留了原本的精度（BF16 或 FP32）；
  • Master weights、weight gradients 和 optimizer states 都是在高精度保存的。

提升混合精度框架的效果

  1. 细粒度的量化
    • 之前的方法是对整个 tensor 进行 scale，把绝对值最大的值 scale 到 FP8 可以表示的范围内，但是这样就会导致异常值会影响到整个 tensor；
    • DeepSeek v3 提出了在更细粒度上进行 scale，比如在 1×1281\times 1281×128 或 128×128128\times 128128×128 的尺度上，这样即使出现了异常值，异常值的影响范围也会更小。
  2. 提升累加精度
    • 在低精度下执行累加会导致精度损失；
    • 我对于论文里描述的理解是计算到一定程度就切到高精度执行累加，保证精度损失不会太多。
  3. Mantissa over Exponents
    • 采用 4-bit 做指数，3-bit 做尾数。
  4. Online Quantization
    • In order to ensure accurate scales and simplify the framework, we calculate the maximum absolute value online for each 1×1281\times 1281×128 activation tile or 128×128128\times 128128×128 weight block.

低精度存储和通信

  • AdamW 的 first moments 和 second moments 采用 BF16 而不是 FP32，不过 master weights 和 gradients 依然采用 FP32；
  • 用于 linear 层 bp 的 activation 是用 FP8 保存的；
  • 把 up-projection 之前的 activation 量化到 FP8，然后进行 dispatch 操作。

预训练

数据

  • 相比 DeepSeek-v2 增加了数据集中数学和编程相关数据的比例，同时扩展了更多的语言；
  • 最终训练数据包含 14.8T tokens；
  • 采用了与 DeepSeek_Coder-v2 类似的 FIM 策略：<∣fim_begin∣>fpre<∣fim_hole∣>fsuf<∣fim_end∣>fmiddle<∣eos_token∣><|fim\_begin|>f_{pre}<|fim\_hole|>f_{suf}<|fim\_end|>f_{middle}<|eos\_token|><∣fim_begin∣>fpre​<∣fim_hole∣>fsuf​<∣fim_end∣>fmiddle​<∣eos_token∣>，但 FIM 的比例只有 0.1；
  • 采用 Byte-level BPE 生成了一个 128K tokens 的词表。

超参数

模型超参数

  • 61 层 transformer 层，hidden dimension 7168；
  • 所有可学习参数随机初始化，标准差为 0.006；
  • MLA 中，attention 模块的 head 数量为 128，dh=128d_h=128dh​=128，压缩之后的 dcd_cdc​ 是 512，query 压缩之后的维度是 1536，dhRd_h^RdhR​ 是 64；
  • FFN 前 3 层没有用 MoE，后面都用了 MoE；
  • 每个 MoE 层包含 1 个共享专家和 256 个 routed experts，专家的 hidden dimension 是 2048；
  • 每个 token 会激活 8 个 routed experts（也就是每个 token 一共激活 9 个 experts），每个 token 最多被发送到 4 个节点上；
  • MTP 的 module 数设为 1；
  • Compressed latent vectors 和 multiplies additional scaling factors at the width bottlenecks 之后加入了 RMS Norm 以稳定训练；
  • DeepSeek-v2 总共有 671B 参数，但是每个 token 只会激活 37B。

训练超参数

  • Optimizer：AdamW，β1=0.9,β2=0.95\beta_1=0.9,\beta_2=0.95β1​=0.9,β2​=0.95，weight_decay=0.1；
  • Maximum sequence length 4K，预训练只在所有语料上训了一轮（14.8T）；
  • 前 2000 step 学习率从 0 线形增加到 2.2×10−42.2\times 10^{-4}2.2×10−4，在 10T 数据上保持 2.2×10−42.2\times 10^{-4}2.2×10−4，然后遵循 cosine 曲线在 4.3T 数据上降低到 2.2×10−52.2\times 10^{-5}2.2×10−5，在 333B 数据上保持，然后在剩余数据上的学习率为 7.6×10−67.6\times 10^{-6}7.6×10−6；
  • Gradient clip 设为 1.0；
  • 在前 496B token 的训练过程中把 batch size 从 3072 逐步增加到 15360。

长文本

  • 采用 YaRN，在 32K 和 128K 分别训练 1000 步，32K 时 batch size 为 1920，128K 时 batch size 480；
  • 学习率为 7.6×10−67.6\times 10^{-6}7.6×10−6。

测试结果

后训练

SFT

数据

  • 总共包含 1.5M 数据；
  • Reasoning 数据：对于 reasoning 数据采用了利用 R1 模型辅助生成训练数据的方法。首先对于每个 domain，训练一个 expert model，expert model 在进行 sft 训练时使用的数据包含两种，一种是原始数据 <problem, original response><problem,\ original\ response><problem, original response>，一种是模型生成的数据 <system prompt, problem, R1 response><system\ prompt,\ problem, \ R1\ response><system prompt, problem, R1 response>。在经过 RL 之后，expert model 就可以输出包含类似于 R1 的思考过程的回答。用 expert model 生成的数据被用作 SFT 训练；
  • 非 reasoning 数据：用 DeepSeek-v2.5 生成回答后用人工审核。

训练设置

  • 训 2 个 epoch；
  • Cosine decay learning rate scheduling，从 5×10−65\times 10^{-6}5×10−6 降低到 1×10−61\times 10^{-6}1×10−6；
  • 训练时每个 sequence 由多个 sample 组成，在 sample 之间加入 mask 保证这些 sample 不会互相影响。

RL Reward Model

  • 对于可以根据 rule 判断的问题，采用 rule based RM，例如：数学题还有 leetcode 的问题；
  • 对于无法根据 rule 判断的问题，采用从 sft checkpoint 训出来的 reward model。

模型整体的技术和之前一代比较像，引入了新的 MTP 技术。同时花了比较大的篇幅讲解 Infra 上面的改动，但是这部分目前我还不怎么了解，所以看得糊里糊涂的，如果有机会的话会以后可以翻出来补上。

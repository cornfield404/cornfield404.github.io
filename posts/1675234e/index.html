题目： DeepSeek-VL: Towards Real-World Vision-Language Understanding
链接： 2403.05525
代码： GitHub - deepseek-ai/DeepSeek-VL: DeepSeek-VL: Towards Real-World Vision-Language Understanding
单位： DeepSeek-AI

本文要点

  1. 数据集构造；
  2. 模型结构；
  3. 训练策略。

之前方法的缺陷

  1. 在 SFT 上分配了过多的计算资源，相对而言，忽视了 pretrain 的重要性；
  2. SFT 数据集主要由 academic dataset 构成，在 benchmark 上取得了比较好的结果，但是在实际使用中使用体验有偏差；
  3. 图像分辨率不够，对于需要高分辨率的场景（例如 OCR 或者小物体识别）无法处理；
  4. 对于模型语言能力没有有效保留。

数据集

Vision-Language pretraining Data

预训练数据主要由以下几类构成：

  1. 图像文本交错的数据；
  2. Image caption：成对的图像文本数据；
  3. 图表数据：使模型具有理解图表的能力；
  4. 代码数据：使模型有从图像中提取代码的能力，由图像和代码片段数据对构成；
  5. 文档 OCR：由 arxiv articles 以及 e-books and educational materials 两部分构成；
  6. Scene text OCR：使模型具有从图像中提取文本的能力；
  7. 纯文本数据：采用了 DeepSeek_LLM 的数据集，使模型的语言能力能够保持。

Supervised Fine-tuning Data

SFT 数据主要有以下几部分构成：

  1. 开源 gpt4v 数据集；
  2. 预训练图表数据集的部分数据；
  3. UI Code 数据集；
  4. In-house instruction-tuning dataset，收集了 GPT-4V 和 Gemini 的真实 case；并按照下表对 case 进行了分类和整理：
  5. 纯文本数据，来自 DeepSeek_LLM。

模型

Hybrid Vision Encoder

仅用 clip 的问题

  1. 因为 clip 模型的训练方式，可能有些图像看起来差距很大但是 clip 之后的编码很接近；
  2. Clip 模型的输入分辨率通常比较低，因此遇到 OCR 之类需要高分辨率的任务就会有问题。

解决方法

采用两个模型，一个的输入为低分辨率图像，另一个的输入为高分辨率图像：

  1. SAM-B：输入图像尺寸为 1024×10241024\times 10241024×1024，输出 feature 尺寸为 64×64×25664\times 64\times 25664×64×256；
  2. SigLIP-L：输入图像尺寸为 384×384384\times 384384×384。

Vision-Language Adaptor

采用两层 MLP。一层用于分别处理 SAM-B 和 SigLIP-L 的特征，SAM-B 特征首先插值到 96×96×25696\times 96\times 25696×96×256，然后用两层步长为 2 的卷积处理，得到的 feature map 尺寸为 24×24×102424\times 24\times 102424×24×1024，然后 reshape 到 576×1024576\times 1024576×1024，这样就可以和 SigLIP-L 生成的特征拼起来得到 576×2048576\times 2048576×2048 的 feature map。然后用另一层 MLP 把视觉 feature 和 LLM feature 对齐。

语言模型

采用了 DeepSeek_LLM。

训练流程

仅训练 adaptor

  • 训练数据包括 1.25 million 成对的 caption 数据以及 2.5 million OCR 数据；
  • 在该阶段继续扩大数据规模没什么用，无法对性能继续提升。

Adaptor 与 LLM 共同训练

  • 该阶段的目的是使 LLM 能够理解多模态输入；
  • 直接用多模态数据训练会导致语言性能严重下降，因此实际训练中在该阶段采用了语言数据:多模态数据=7:3的比例对模型进行训练；
  • 在该阶段中发现 1.3B 模型的指标大幅波动，导致无法用 1.3B 上的实验推测更大规模模型上的结果，为了解决该问题采取了两个措施：
    1. 把测试方式由 multi choice 改成根据答案的 PPL 选择 answer；
    2. 引入一定比例的 sft 数据作为训练数据。

SFT

  • 该阶段只有 SAM-B 不参与训练，模型其他部分均参与训练

训练参数

一些有效的 trick

  1. 训练时保证每个 batch 仅包含纯文本数据或多模态数据可以在不损失性能的情况下加速模型训练，主要是纯文本数据速度比较快，多模态数据更慢，如果混在一个 batch 里的话纯文本数据跑完还需要等多模态数据；
  2. 最开始采用全文本数据，然后逐步把纯文本数据的比例降低到 0.7 可以稳定训练。

个人总结

比较重要的点还是数据和模型两方面：

  1. 数据：收集了多个来源的图像文本对，同时在数据集里混入了纯文本数据，以保证模型在语言任务上的性能；
  2. 模型上采用了混合尺度的视觉编码器，用 SAM-B 处理高分辨率图像，SigLIP-L 处理低分辨率图像，得到不同分辨率下的图像特征。
